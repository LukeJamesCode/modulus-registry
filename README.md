# modulus-registry

The curated module registry for [Modulus](https://github.com/LukeJamesCode). Modulus
reads `index.json` from this repo to power the panel's **Modules → Browse marketplace**
and `modulus mod install <name>`. Each entry pins a tarball by `sha256`, so a swapped or
tampered tarball fails closed on the client.

Modulus only ever **reads** this registry. The client side (fetch, sha256 verify, strict
extract, consent, install) lives in `src/core/registry.ts` and `src/core/installer.ts`.

## Layout

```
modules/<name>/      vendored module source (one folder per published module)
index.json           generated by CI on each release — the file Modulus fetches
.github/workflows/   publish.yml: packs modules, cuts the release, regenerates index.json
```

Modules are **vendored** here (the Modulus app repo is developed locally and has no public
remote to pull from). To publish a new or updated module, copy its folder from
`Modulus/modules/<name>/` into `modules/<name>/` here, then cut a release.

## Publishing a release

1. Make sure `modules/<name>/manifest.json` has the right `version` and an honest
   `permissions` block (that block is exactly what the user consents to on install).
2. Commit to `main`.
3. Push a version tag:
   ```sh
   git tag v1.0.0 && git push origin v1.0.0
   ```
4. The `publish` workflow then:
   - packs each `modules/*/` into a strict-ustar `.tgz` (sha256-pinned),
   - attaches the tarballs to the GitHub Release for that tag,
   - regenerates `index.json` and commits it back to `main`.

All modules are republished under one release per tag; the tarball URLs reference that
tag. Re-tag (e.g. `v1.0.1`) to publish again.

## Tarball rules (the strict extractor)

`installer.ts` is deliberately strict, and the CI packs to match:

- **Plain `ustar`** only — no pax/gnu long-name entries, so every path stays ≤ 100 bytes
  (keep module names short and layouts shallow).
- `manifest.json` under a single `./<name>/` prefix; `manifest.name` **must equal** the
  index entry's `name`.
- No absolute paths, no `..`, no symlinks. Caps: ~50 MB compressed, ~200 MB unpacked.
- Do **not** use `npm pack` — it emits pax headers and a `package/` prefix the extractor
  rejects. CI uses `tar --format=ustar -C modules <name>`.

## Heavy modules

`modulus-browser` (Playwright) and `modulus-discord` (discord.js) ship source only; their
third-party npm dependencies are installed separately when the module is enabled (see each
module's README). They install and consent like any other module, but won't load until
those deps are present.
